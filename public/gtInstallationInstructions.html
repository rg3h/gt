gt Installation Instructions
An extensible set of tools to simplify git and github repos
version 1.0.0

Installation

You will need to install the following apps:
  • git (you will also need to set up a .gitconfig and .gitignore)
  • create a github account
  • gh for github
  • zsh and sed (on windows I use cygwin)
  • download the gt.zip file and unzip it.
  • Move the gt directory into your execution path (like your bin/ directory)
    (note for windows you might need to edit your PATH environment variable)

Commands

gt supports the following commands:

gt

┌────────────────────────────────────────────────────────────────────┐
│ gt (v1.0.0): an easy-to-use tool for git and github                │
│ example: gt addRepo myNewRepo --private                            │
├────────────────────────────────────────────────────────────────────┤
│ help      | --help     | -h | -? .. show this help                 │
│ addRepo   | makeRepo   | ar | mr... add a new repo                 │
│ delRepo   | removeRepo | rr | dr... delete a repo from github      │
│ listRepos | lr..................... list your gitHub repos         │
│ version   | -v..................... show the version (v1.0.0)      │
└────────────────────────────────────────────────────────────────────┘

addRepo (ar)

┌────────────────────────────────────────────────────────────────────┐
│ gt [addRepo | ar] remoteRepoName [localRepoName] [--private]       │
├────────────────────────────────────────────────────────────────────┤
│ adds a github remote repo and a local repo directory               │
│   --help | -h | -? issues this help                                │
│   --private adds a private repo (default is public)                │
│                                                                    │
│ examples:                                                          │
│  gt addRepo myRepo    # add a local and remote repo named myRepo   │
│  gt addRepo myRepo .  # add a remote w/ local repo in current dir  │
│  gt addRepo myRepo --private                                       │
│  gt addRepo myRepo myLocalRepoName                                 │
│  gt addRepo myRepo myLocalRepoName --private                       │
└────────────────────────────────────────────────────────────────────┘

delRepo (dr)

┌────────────────────────────────────────────────────────────────────┐
│ gt [delRepo | dr] remoteRepoName [--yes]                           │
├────────────────────────────────────────────────────────────────────┤
│ deletes the remote repo but does NOT delete the local repo dir     │
│   --help | -h | -? issues this help                                │
│   --yes skips the confirmation check                               │
│                                                                    │
│ examples:                                                          │
│  gt delRepo myRepo  # deletes the remote repo named myRepo         │
│  gt delRepo myRepo --yes # deletes remote repo w/o confirming      │
└────────────────────────────────────────────────────────────────────┘

listRepos (lr)

┌────────────────────────────────────────────────────────────────────┐
│ gt [listRepos | lr] [--sort [date|name|private|public]]            │
├────────────────────────────────────────────────────────────────────┤
│ lists up to 100 of your remote repos in a table                    │
│   --help | -h | -?  issues this help                               │
│   --sort | -s [date|name|private] sorts by one of these fields     │
│                                                                    │
│ examples:                                                          │
│  gt listRepos         # lists your repos sorted by date            │
│  gt lr --sort name    # lists your repos sorted by name            │
│  gt lr --sort private # lists your repos with private ones first   │
│  gt lr --sort public  # lists your repos with public ones first    │
└────────────────────────────────────────────────────────────────────┘

Extensible zsh Architecture

zsh is a pain. But I've tried to write it so that it is modular. The structure is: 
    gt.sh
    gtAddRepo.sh
    gtDelRepo.sh
    gtListRepos.sh
    modules/
      gtCharCodes.sh   -- TTY character controls to create the ascii boxes
      gtDebug.sh       -- debugging that can be turned on to see details
      gtGetOwner.sh    -- getOwner is a gh command used internally by gt
      gtPrintBox.sh    -- prints the white and red boxes around text
      gtStatusCodes.sh -- the status codes as messages
      gtUtil.sh        -- utility functions
